# 安全基线约束规则

在进行代码生成、接口设计、数据处理时，必须遵循以下安全规则：

## 规则数据源

参考 #[[file:skills/security-baseline/security-rules.yaml]] 中定义的安全规范。

## 安全红线（任何场景不可违反）

1. 密码存储必须使用 bcrypt 或 argon2 单向哈希，禁止 MD5、SHA1、明文存储
2. 数据库查询必须使用参数化查询或 ORM，禁止拼接 SQL 字符串
3. 禁止将用户输入直接嵌入系统命令、SQL 语句、文件路径
4. 禁止在代码仓库中存储密钥、密码、Token 等凭证，必须通过环境变量注入
5. 生产环境错误响应禁止返回堆栈信息、SQL 语句、文件路径等技术细节
6. 生产环境 CORS 禁止使用 `Access-Control-Allow-Origin: *`
7. 日志中禁止输出密码、Token、密钥等敏感信息原文
8. 随机数生成必须使用加密安全的生成器（Python 用 secrets 模块，禁止用 random）
9. 所有用户输入必须在服务端校验，前端校验不可替代服务端校验

## 推荐规则

1. JWT access_token 过期时间设置为 15-60 分钟，refresh_token 7-30 天
2. 登录失败不暴露具体原因（不区分"用户不存在"和"密码错误"）
3. 登录失败达到阈值后锁定账户或增加延迟
4. 所有输出到 HTML 的内容必须转义，API 响应设置 Content-Type: application/json
5. 设置安全响应头：X-Content-Type-Options、X-Frame-Options、Strict-Transport-Security
6. 生产环境必须使用 HTTPS，TLS 版本不低于 1.2
7. 高敏感数据（身份证号、银行卡号）加密存储，中敏感数据（手机号、邮箱）脱敏展示
8. 依赖版本必须锁定，CI 构建时自动扫描依赖漏洞
9. 关键接口（登录、注册、短信发送）实施速率限制
10. 列表接口必须分页，单页最大条数限制

## 场景匹配

- 实现登录/注册 → 检查密码哈希、JWT 配置、登录防护规则
- 编写数据库查询 → 检查 SQL 注入防护，必须参数化
- 处理用户输入 → 检查输入验证、长度限制、格式校验
- 配置 API 响应 → 检查错误信息泄露、安全响应头、CORS 配置
- 处理文件上传 → 检查路径遍历防护、文件类型和大小限制
- 记录日志 → 检查敏感数据脱敏
- 引入新依赖 → 检查依赖安全性和版本锁定
