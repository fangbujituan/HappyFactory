# 安全基线 Skill

## 目的

在自动编码过程中，确保 AI 生成的代码遵循安全最佳实践，避免引入常见安全漏洞。覆盖认证授权、输入验证、数据保护、密钥管理、HTTP 安全等关键领域。

## 使用方式

本 skill 作为 steering 规则自动生效，当 Kiro 在以下场景中工作时会参考此安全基线：

- 实现用户认证和授权逻辑
- 处理用户输入和数据校验
- 编写数据库查询
- 配置 HTTP 响应头和 CORS
- 管理密钥和敏感配置
- 编写日志记录逻辑
- 设计 API 接口

## 安全红线（不可违反）

1. 密码必须使用 bcrypt 或 argon2 哈希存储，禁止明文或 MD5/SHA1
2. 禁止拼接 SQL 字符串，必须使用参数化查询或 ORM
3. 禁止在代码仓库中存储密钥和凭证
4. 生产环境禁止暴露堆栈信息、SQL 语句等技术细节
5. 生产环境禁止 CORS 使用通配符 *
6. 日志中禁止输出密码、Token 等敏感信息原文

## 维护

编辑 `security-rules.yaml` 来调整安全规则。修改 `level` 字段控制规则的强制程度：
- `required`：安全红线，必须遵循
- `preferred`：推荐遵循，显著降低风险
- `optional`：可选，视安全等级决定
