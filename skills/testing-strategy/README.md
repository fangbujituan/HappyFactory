# 测试规范 Skill

## 目的

在自动编码过程中，确保 AI 生成的测试代码符合团队的测试规范和质量标准。覆盖测试分层、命名、覆盖率、Mock 策略、数据管理等方面。

## 使用方式

本 skill 作为 steering 规则自动生效，当 Kiro 在以下场景中工作时会参考此测试规范：

- 为新功能编写测试用例
- 生成测试框架脚手架
- 设计测试策略（spec 的 design 阶段）
- 配置 CI 中的测试流程

## 核心规范

1. 遵循测试金字塔：单元测试多、集成测试适中、E2E 测试少
2. 测试命名：test_<方法名>_<场景>
3. 测试结构：Arrange-Act-Assert (AAA) 模式
4. 每个测试只测一个行为，测试之间必须独立
5. 测试数据与逻辑分离，使用 YAML 文件 + parametrize
6. 所有测试必须标记类型（@pytest.mark.api 等）

## 维护

编辑 `testing-rules.yaml` 调整规则，修改 `level` 控制强制程度。
